Workflow

What happens after a user clicks upload?

01

Validate the payload

The API checks the file type, MIME, payload size, and video duration before any inference work starts.

02

Run the detector

Images are scored directly. Videos are sampled frame by frame and aggregated into a single final verdict.

03

Return an auditable response

The client receives the verdict, fake probability, model name, processing time, and a request identifier for debugging.

FAQ

Direct answers for public users.

Is this perfect proof that media is fake?

No. VeriLens is a detection aid. It should be used alongside source checks, metadata review, reverse image search, and context verification.

Why are uploads capped?

Free hosting has real limits. The caps keep the public demo stable and reduce abuse.

Why do videos take longer than images?

Videos are decoded, sampled across multiple frames, and then aggregated into a final score. That is inherently heavier than a single image pass.
